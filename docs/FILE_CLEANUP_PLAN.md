# File Cleanup Plan

## Purpose

This document lists cleanup work needed to make Tetherpoint easier to inspect, safer to edit, and easier for agents or engineers to inherit.

The goal is not cosmetic cleanup.

The goal is to make failure smaller, make ownership visible, and reduce cognitive load.

## Core cleanup rule

Each file should have one clear job.

Each layer should leave the next layer a clean handoff.

If a file or layer is carrying too many responsibilities, split it before adding new behavior.

# Agent task categories

## Safe agent tasks

- create docs
- create new small files
- move sample text to a sample file
- search for imports/usages
- export an existing helper
- add explicit comments or labels
- run verification and report errors

## Medium-risk agent tasks

- update imports across files
- move shared helpers out of a large file
- remove unused alternate surfaces after import verification
- update tests for exact known failures

## High-risk tasks

- whole-file replacement of `ReceiptWorkspace.tsx`
- broad UI refactor
- backend behavior changes
- OpenAPI/schema changes
- pipeline order changes
- source/retrieval behavior changes
- deleting alternate workspaces before usage is verified

High-risk tasks need human-guided review or smaller staged steps.

# Success condition

A new engineer or agent should be able to open the repo and answer:

- what file owns this behavior?
- what layer produced this data?
- what depends on this output?
- what is safe to edit?
- what should not be touched without scope?
- where did this change enter the repo?

If those answers are not visible, cleanup is not done.
